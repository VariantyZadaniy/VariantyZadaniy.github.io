Вариант № 3

Модуль № 1: Разработка, администрирование и защита баз данных

Задание:

Центральная городская библиотека им. А.С. Пушкина содержит обширный фонд книг различных жанров и авторов. Для эффективного управления фондом и обслуживания читателей важно иметь актуальную информацию о наличии книг, их состоянии и авторах.

Разработать подсистему для работы с книгами и авторами, обеспечивающую следующий функционал:

  • Просмотр списка книг
  • Добавление/редактирование данных о книгах
  • Просмотр списка авторов
  • Учет связи книг с авторами

На основе описания предметной области (Приложение 1) необходимо создать базу данных в выбранной СУБД для разрабатываемой системы. Обязательна 3 нормальная форма с обеспечением ссылочной целостности. При разработке базы данных обратить внимание на согласованную схему именования, создать необходимые первичные и внешние ключи.

На данном этапе достаточно создать таблицы, поля и связи, относящиеся к разрабатываемой подсистеме (книги, авторы, связь книг и авторов).

Примечание: Одна книга может иметь несколько авторов (соавторство), и один автор может быть связан с несколькими книгами.

Требования к результатам Модуля 1:

  1. Импорт данных: Заказчик системы предоставил файлы с данными (с пометкой import в ресурсах) для переноса в новую систему. Необходимо подготовить данные файлов для импорта и загрузить в разработанную базу данных.
  2. Скрипт БД: Сохранить полученные результаты: создать SQL-скрипт, содержащий DDL (создание структуры: таблиц, ключей) и DML (импортированные данные) для воссоздания базы данных. Скрипт должен быть исполняемым.
Модуль № 2: Разработка модулей программного обеспечения для компьютерных систем

Задание:

Сформировать алгоритм подсистемы для работы с книгами библиотеки. Разработать алгоритм функции определения книг, требующих восстановления или замены.

Алгоритмы реализовать в виде кода программного продукта средствами любой среды разработки и языка программирования из доступных (например, C#, Java, Python).

Требования к интерфейсу и оформлению:

  • Компоненты системы должны иметь единый согласованный внешний вид, соответствующий руководству по стилю (Приложение 2).
  • Заголовок главного окна (страницы) должен соответствовать назначению (например, "Учет книг - Библиотека им. Пушкина").
  • Установить иконку приложения (предоставлена в ресурсах), если это реализуемо в рамках платформы.
  • Разместить логотип библиотеки (предоставлен в ресурсах) на главной форме. Логотип не искажать.
  • Использовать шрифт Merriweather.
  • Цветовая схема:
    • Основной фон: #F8F5F1 (Светло-бежевый)
    • Дополнительный фон: #8C9EAD (Серо-голубой)
    • Акцентирование внимания: #3A5F8A (Темно-синий)

Требования к коду:

  • Идентификаторы (имена переменных, функций, классов) должны отражать их назначение.
  • Соблюдать соглашения об именовании:
    • CamelCase (для C# и Java)
    • snake_case (для Python)
  • Допустимо использование не более одной команды в строке.
  • Код должен быть хорошо документирован комментариями.

Функциональные требования к модулю:

  1. Вывод списка книг: Разработать программный модуль (главную форму/окно) для учета книг. Реализовать вывод списка книг, хранящихся в базе данных, согласно следующему описанию:
    • Отображать таблицу/список со столбцами: Название книги, Автор(ы), Жанр, Год издания, Состояние, Статус, Оценка износа, Требуется замена.
    • Предусмотреть возможность сортировки по любому столбцу (опционально, но желательно).
  2. Определение книг, требующих восстановления: Для каждой книги в столбце Требуется замена необходимо выводить соответствующее значение.
    • Алгоритм расчета:
      1. Если Состояние = 'Отличное': Требуется замена = "Нет"
      2. Если Состояние = 'Хорошее' и Оценка износа < 60: Требуется замена = "Нет"
      3. Если Состояние = 'Хорошее' и Оценка износа >= 60: Требуется замена = "Восстановление"
      4. Если Состояние = 'Удовлетворительное': Требуется замена = "Восстановление"
      5. Если Состояние = 'Плохое' или Оценка износа >= 80: Требуется замена = "Замена"
      6. Если Статус = 'Утеряна': Требуется замена = "Замена"
    • Значение должно отображаться в соответствующем столбце.
  3. Подключение БД: Созданную на Модуле 1 базу данных подключить к данному приложению. Список книг на главной форме должен отображать актуальные данные из БД.
  4. Отладка и тестирование: Выполнить отладку и тестирование модуля для проверки функциональности. Приложение должно:
    • Корректно запускаться и закрываться.
    • Корректно отображать данные из БД.
    • Корректно определять необходимость восстановления/замены для всех книг в списке.
    • Не завершаться аварийно при стандартных операциях.

Необходимые приложения

Приложение 1: Описание предметной области (Управление библиотекой)

Описание предметной области

Центральная городская библиотека им. А.С. Пушкина обслуживает более 50 000 читателей и содержит фонд из более чем 200 000 книг. Библиотека предоставляет услуги выдачи книг на дом, читальные залы, электронный каталог и мероприятия.

Разрабатываемая система включает подсистемы:

  • Учет книг и фондов
  • Управление авторами
  • Обслуживание читателей
  • Учет выдачи и возврата книг
  • Управление филиалами

Основные составляющие предметной области:

1. Книги: ISBN, Название, Жанр, Год издания, Издательство, Количество страниц, Аннотация, Язык, Состояние (Отличное, Хорошее, Удовлетворительное, Плохое), Статус (В фонде, Выдана, На реставрации, Утеряна), Оценка износа (0-100%), Дата поступления, Стоимость, Расположение (стеллаж, полка).

2. Авторы: ФИО, Дата рождения, Дата смерти (если применимо), Страна, Биография, Основные жанры, Фотография.

3. Читатели: ФИО, Номер читательского билета, Адрес, Телефон, Email, Дата регистрации, Категория (взрослый, ребенок, пенсионер, студент).

4. Выдача книг: Читатель, Книга, Дата выдачи, Планируемая дата возврата, Фактическая дата возврата, Статус (активна, возвращена, просрочена), Штраф (если применимо).

5. Библиотекари: ФИО, Должность, Контактные данные, График работы, Филиал.

6. Филиалы библиотеки: Название, Адрес, Телефон, Режим работы, Заведующий, Количество книг в фонде.

7. Жанры: Название жанра, Описание, Количество книг в данном жанре.

8. Восстановление книг: Книга, Причина восстановления, Дата принятия в работу, Дата завершения, Стоимость работ, Исполнитель, Статус.

Приложение 2: Руководство по стилю

Все экранные формы пользовательского интерфейса должны иметь заголовок. На главной форме должен быть установлен логотип библиотеки (представлен в ресурсах). Логотип не искажать: не менять изображение, пропорции изображения, цвет.

Для приложения должна быть установлена иконка (дана в ресурсах), если это реализуемо в рамках платформы.

Использовать шрифт Merriweather.

Цветовая схема:
