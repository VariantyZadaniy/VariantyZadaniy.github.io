Вариант № 1

Модуль № 1: Разработка, администрирование и защита баз данных

Задание:

Компания "Дубовый Стандарт" занимается производством корпусной мебели и использует различные типы материалов (древесина, фурнитура, клей, лак и т.д.). Для эффективной работы производства и своевременного выполнения заказов критически важно иметь актуальную информацию об остатках материалов на складе.

Разработать подсистему для работы с материалами, которые используются для производства мебели, обеспечивающую следующий функционал:

  • Просмотр списка материалов
  • Добавление/редактирование данных о материале
  • Просмотр списка поставщиков материала

На основе описания предметной области (Приложение 1) необходимо создать базу данных в выбранной СУБД для разрабатываемой системы. Обязательна 3 нормальная форма с обеспечением ссылочной целостности. При разработке базы данных обратить внимание на согласованную схему именования, создать необходимые первичные и внешние ключи.

На данном этапе нет необходимости воспроизводить все сущности предметной области, достаточно создать таблицы, поля с подходящими типами данных и связи, непосредственно относящиеся к разрабатываемой подсистеме (материалы, поставщики, связь материалов и поставщиков) и ее функционалу.

Требования к результатам Модуля 1:

  1. Импорт данных: Заказчик системы предоставил файлы с данными (с пометкой import в ресурсах) для переноса в новую систему. Необходимо подготовить данные файлов для импорта и загрузить в разработанную базу данных.
  2. Скрипт БД: Сохранить полученные результаты: создать SQL-скрипт, содержащий DDL (создание структуры: таблиц, ключей) и DML (импортированные данные) для воссоздания базы данных. Скрипт должен быть исполняемым.
Модуль № 2: Разработка модулей программного обеспечения для компьютерных систем

Задание:

Сформировать алгоритм подсистемы для работы с материалами. Разработать алгоритм функции расчета стоимости минимально необходимой партии материала.

Алгоритмы реализовать в виде кода программного продукта средствами любой среды разработки и языка программирования из доступных (например, C#, Java, Python).

Требования к интерфейсу и оформлению:

  • Компоненты системы должны иметь единый согласованный внешний вид, соответствующий руководству по стилю (Приложение 2).
  • Заголовок главного окна (страницы) должен соответствовать назначению (например, "Учет материалов - Мебельное производство").
  • Установить иконку приложения (предоставлена в ресурсах), если это реализуемо в рамках платформы.
  • Разместить логотип компании "Дубовый Стандарт" (предоставлен в ресурсах) на главной форме. Логотип не искажать.
  • Использовать шрифт Comic Sans MS.
  • Цветовая схема:
    • Основной фон: #FFFFFF (Белый)
    • Дополнительный фон: #ABCBCE (Светло-серо-голубой)
    • Акцентирование внимания: #546F94 (Темно-синий)

Требования к коду:

  • Идентификаторы (имена переменных, функций, классов) должны отражать их назначение.
  • Соблюдать соглашения об именовании:
    • CamelCase (для C# и Java)
    • snake_case (для Python)
  • Допустимо использование не более одной команды в строке.

Функциональные требования к модулю:

  1. Вывод списка материалов: Разработать программный модуль (главную форму/окно) для учета материалов. Реализовать вывод списка материалов, хранящихся в базе данных, согласно следующему описанию:
    • Отображать таблицу/список со столбцами: Наименование материала, Тип материала, Единица измерения, Кол-во на складе, Минимальное кол-во, Кол-во в упаковке, Цена за единицу, Стоимость мин. партии.
    • Предусмотреть возможность сортировки по любому столбцу (опционально, но желательно).
  2. Расчет стоимости минимальной партии: Для каждого материала в столбце Стоимость мин. партии необходимо выводить рассчитанное значение.
    • Алгоритм расчета:
      1. Если Кол-во на складе >= Минимальное кол-во: Стоимость мин. партии = 0.00 (запаса достаточно).
      2. Если Кол-во на складе < Минимальное кол-во:
        • Рассчитать Дефицит = Минимальное кол-во - Кол-во на складе.
        • Рассчитать Минимальный объем закупки = ceil(Дефицит / Кол-во в упаковке) * Кол-во в упаковке. (Округление вверх до целого числа упаковок).
        • Рассчитать Стоимость мин. партии = Минимальный объем закупки * Цена за единицу.
    • Стоимость минимальной партии должна отображаться с двумя знаками после запятой (формат денег).
    • Стоимость не может быть отрицательной. При достаточном запасе выводить 0.00.
  3. Подключение БД: Созданную на Модуле 1 базу данных подключить к данному приложению. Список материалов на главной форме должен отображать актуальные данные из БД.
  4. Отладка и тестирование: Выполнить отладку и тестирование модуля для проверки функциональности. Приложение должно:
    • Корректно запускаться и закрываться.
    • Корректно отображать данные из БД.
    • Корректно рассчитывать и отображать стоимость минимальной партии для всех материалов в списке.
    • Не завершаться аварийно при стандартных операциях.

Необходимые приложения

Приложение 1: Описание предметной области (Мебельное производство)

Описание предметной области

Производственная компания «Дубовый Стандарт» выпускает корпусную мебель (шкафы, кухни, тумбы, столы). Компания занимается проектированием, производством и реализует свою продукцию через собственные салоны и партнеров-дилеров.

Разрабатываемая система включает подсистемы:

  • Каталог продукции (мебельные изделия)
  • Склад и материалы
  • Производственные заказы
  • Сотрудники
  • Дилеры (Партнеры)

Основные составляющие предметной области:

1. Дилеры (Партнеры): Тип (салон мебели, интернет-магазин, оптовик), наименование компании, юридический адрес, ИНН, ФИО директора/контактного лица, телефон, email, логотип, уровень скидки (зависит от общего объема закупок за год). Дилеры покупают продукцию со скидкой. Взаиморасчеты в рублях.

2. Менеджеры по продажам: Работа с дилерами: регистрация, прием заказов, согласование спецификаций, контроль оплаты, формирование предложений на основе истории закупок дилера.

3. Производственный заказ: Формируется на основе заказа от дилера или собственного салона. Содержит спецификацию изделий (артикул, количество), требования к материалам (фактура, цвет), срок изготовления. После подтверждения и предоплаты запускается в производство. Управляется мастером цеха.

4. Сотрудники: ФИО, должность (менеджер, мастер цеха, сборщик, станочник и т.д.), контактные данные, дата приема.

5. Мастер цеха: Отвечает за планирование производства, заказ материалов у поставщиков, контроль качества продукции и материалов, выдачу заданий сборщикам.

6. Материалы (сырье и комплектующие): Тип (ЛДСП, МДФ, массив дерева, кромка, клей, лак, крепеж, ручки, петли, направляющие и т.д.), Наименование (например, "ЛДСП 18мм Дуб Сонома"), Поставщик(и), Количество в упаковке (листов, пачек, штук, кг, л), Единица измерения для учета на складе (лист, шт, кг, л), Описание (цвет, размер листа), Стоимость за единицу измерения, Количество на складе, Минимальное допустимое количество (порог остатка), История изменений остатков.

7. Склад материалов: Учет поступлений материалов от поставщиков, резервирование материалов под конкретные производственные заказы, отпуск материалов в цех, инвентаризация. Предоставление информации о текущих остатках материалов.

8. Поставщики материалов: Тип (производитель, дистрибьютор), Наименование, ИНН, Контактные данные, Основные поставляемые материалы, История поставок (дата, материал, количество, стоимость).

9. Продукция (Мебельные изделия): Артикул, Тип изделия (шкаф, кухня, тумба, стол), Наименование модели, Описание (габариты, цвет), Изображение, Стоимость производства, Розничная цена, Цена для дилера, Габариты упаковки, Вес, Спецификация необходимых материалов (Билл материалов - BOM).
