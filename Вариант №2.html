Модуль № 1: Разработка, администрирование и защита баз данных

Задание:

Кофейня "Кофейный Рай" использует различные виды ингредиентов (кофейные зерна, молоко, сиропы, выпечка и т.д.). Для эффективной работы кофейни и своевременного обслуживания клиентов критически важно иметь актуальную информацию об остатках ингредиентов на складе.

Разработать подсистему для работы с ингредиентами, которые используются для приготовления напитков и блюд, обеспечивающую следующий функционал:

  • Просмотр списка ингредиентов
  • Добавление/редактирование данных об ингредиенте
  • Просмотр списка поставщиков ингредиентов

На основе описания предметной области (Приложение 1) необходимо создать базу данных в выбранной СУБД для разрабатываемой системы. Обязательна 3 нормальная форма с обеспечением ссылочной целостности. При разработке базы данных обратить внимание на согласованную схему именования, создать необходимые первичные и внешние ключи.

На данном этапе нет необходимости воспроизводить все сущности предметной области, достаточно создать таблицы, поля с подходящими типами данных и связи, непосредственно относящиеся к разрабатываемой подсистеме (ингредиенты, поставщики, связь ингредиентов и поставщиков) и ее функционалу.

Требования к результатам Модуля 1:

  1. Импорт данных: Заказчик системы предоставил файлы с данными (с пометкой import в ресурсах) для переноса в новую систему. Необходимо подготовить данные файлов для импорта и загрузить в разработанную базу данных.
  2. Скрипт БД: Сохранить полученные результаты: создать SQL-скрипт, содержащий DDL (создание структуры: таблиц, ключей) и DML (импортированные данные) для воссоздания базы данных. Скрипт должен быть исполняемым.
Модуль № 2: Разработка модулей программного обеспечения для компьютерных систем

Задание:

Сформировать алгоритм подсистемы для работы с ингредиентами. Разработать алгоритм функции расчета стоимости минимально необходимой партии ингредиента.

Алгоритмы реализовать в виде кода программного продукта средствами любой среды разработки и языка программирования из доступных (например, C#, Java, Python).

Требования к интерфейсу и оформлению:

  • Компоненты системы должны иметь единый согласованный внешний вид, соответствующий руководству по стилю (Приложение 2).
  • Заголовок главного окна (страницы) должен соответствовать назначению (например, "Учет ингредиентов - Кофейня").
  • Установить иконку приложения (предоставлена в ресурсах), если это реализуемо в рамках платформы.
  • Разместить логотип кофейни "Кофейный Рай" (предоставлен в ресурсах) на главной форме. Логотип не искажать.
  • Использовать шрифт Roboto.
  • Цветовая схема:
    • Основной фон: #F9F5F0 (Светло-бежевый)
    • Дополнительный фон: #D7CCC8 (Светло-коричневый)
    • Акцентирование внимания: #5D4037 (Темно-коричневый)

Требования к коду:

  • Идентификаторы (имена переменных, функций, классов) должны отражать их назначение.
  • Соблюдать соглашения об именовании:
    • CamelCase (для C# и Java)
    • snake_case (для Python)
  • Допустимо использование не более одной команды в строке.

Функциональные требования к модулю:

  1. Вывод списка ингредиентов: Разработать программный модуль (главную форму/окно) для учета ингредиентов. Реализовать вывод списка ингредиентов, хранящихся в базе данных, согласно следующему описанию:
    • Отображать таблицу/список со столбцами: Наименование ингредиента, Тип ингредиента, Единица измерения, Кол-во на складе, Минимальное кол-во, Кол-во в упаковке, Цена за единицу, Стоимость мин. партии.
    • Предусмотреть возможность сортировки по любому столбцу (опционально, но желательно).
  2. Расчет стоимости минимальной партии: Для каждого ингредиента в столбце Стоимость мин. партии необходимо выводить рассчитанное значение.
    • Алгоритм расчета:
      1. Если Кол-во на складе >= Минимальное кол-во: Стоимость мин. партии = 0.00 (запаса достаточно).
      2. Если Кол-во на складе < Минимальное кол-во:
        • Рассчитать Дефицит = Минимальное кол-во - Кол-во на складе.
        • Рассчитать Минимальный объем закупки = ceil(Дефицит / Кол-во в упаковке) * Кол-во в упаковке. (Округление вверх до целого числа упаковок).
        • Рассчитать Стоимость мин. партии = Минимальный объем закупки * Цена за единицу.
    • Стоимость минимальной партии должна отображаться с двумя знаками после запятой (формат денег).
    • Стоимость не может быть отрицательной. При достаточном запасе выводить 0.00.
  3. Подключение БД: Созданную на Модуле 1 базу данных подключить к данному приложению. Список ингредиентов на главной форме должен отображать актуальные данные из БД.
  4. Отладка и тестирование: Выполнить отладку и тестирование модуля для проверки функциональности. Приложение должно:
    • Корректно запускаться и закрываться.
    • Корректно отображать данные из БД.
    • Корректно рассчитывать и отображать стоимость минимальной партии для всех ингредиентов в списке.
    • Не завершаться аварийно при стандартных операциях.

Необходимые приложения

Приложение 1: Описание предметной области (Управление кофейней)

Описание предметной области

Кофейня "Кофейный Рай" специализируется на приготовлении кофейных напитков и десертов. Компания управляет несколькими кофейнями в городе, предлагает напитки на вынос и доставку десертов.

Разрабатываемая система включает подсистемы:

  • Меню напитков и десертов
  • Учет ингредиентов
  • Управление заказами
  • Сотрудники
  • Поставщики

Основные составляющие предметной области:

1. Поставщики: Тип (производитель, дистрибьютор), наименование компании, юридический адрес, ИНН, ФИО контактного лица, телефон, email, логотип, рейтинг надежности. Поставщики доставляют ингредиенты по установленному графику. Взаиморасчеты в рублях.

2. Бариста и менеджеры: Бариста готовят напитки, менеджеры управляют заказами, контролируют остатки ингредиентов и взаимодействуют с поставщиками.

3. Заказ клиента: Формируется на основе выбора клиента. Содержит позиции меню (напитки, десерты), требования к приготовлению (например, "без сахара", "с соевым молоком"). После оплаты передается на выполнение бариста.

4. Сотрудники: ФИО, должность (бариста, менеджер, администратор, уборщик), контактные данные, график работы.

5. Менеджер смены: Отвечает за планирование работы кофейни, заказ ингредиентов у поставщиков, контроль качества продукции и ингредиентов, выдачу заданий сотрудникам.

6. Ингредиенты: Тип (кофейные зерна, молоко, сиропы, сахар, мука, шоколад и т.д.), Наименование (например, "Кофе арабика, Бразилия"), Поставщик(и), Количество в упаковке (кг, л, шт), Единица измерения для учета на складе (кг, л, шт), Описание (сорт, страна происхождения), Стоимость за единицу измерения, Количество на складе, Минимальное допустимое количество (порог остатка), Срок годности.

7. Склад ингредиентов: Учет поступлений ингредиентов от поставщиков, контроль сроков годности, отпуск ингредиентов на кухню, инвентаризация. Предоставление информации о текущих остатках ингредиентов.

8. Позиции меню: Артикул, Тип (напиток, десерт), Наименование, Описание (состав, способ приготовления), Изображение, Себестоимость, Розничная цена, Калорийность, Рецепт (список необходимых ингредиентов с количеством).
